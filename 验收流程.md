### 演示操作（核心！）

**这是验收的重点环节，直接演示 Web 页面：**

```
1. 打开 http://127.0.0.1:5000
2. 点 "Screen vs Battery" 快捷例句
3. 方面词保持 "screen"，模型选 BERT_SPC
4. 点 "🔍 单模型分析" → 展示结果：Positive ✅
5. 将方面词改为 "battery"
6. 点 "⚡ 双模型对比"
```

**此时屏幕上会同时显示：**
- 左侧 LSTM → Positive（**错误！** 电池差却说正面）
- 右侧 BERT_SPC → Negative（**正确！**）
- 底部橙色警告：⚠️ 两个模型预测不一致

### 口头说辞（直接用这个讲）

> "我训练了两种模型做对比：
>
> **LSTM** 是基础循环神经网络，用静态 GloVe 词向量，准确率 68.5%。
> **BERT** 是预训练 Transformer，用 bert-base-uncased，准确率 78.7%，比 LSTM 高了 10 个百分点。
>
> 但准确率不是最重要的——关键是**能力差异**。请看这个例子：'The screen is great but the battery is poor.' ——屏幕好，电池差，两个相反的情感。
>
> **LSTM 的问题**：它把整句话编码成一个向量，不管我说 'screen' 还是 'battery'，结果都一样——它分不出来。
>
> **BERT 怎么做到的**：我把输入格式改成了 '[CLS] 整句 [SEP] 方面词 [SEP]'，通过 Segment Embedding，BERT 知道我现在分析的是哪个方面词，所以能给出不同的答案。
>
> 这就是 BERT 比 LSTM 好的根本原因——**方面词感知能力**。"

### 如果老师追问

| 问题 | 回答 |
|------|------|
| "为什么BERT更好？" | 三个方面：①预训练在大规模语料上，有先验知识；②12层Transformer能捕捉长距离依赖；③Segment Embedding区分句子和方面词 |
| "LSTM为什么差？" | GloVe是静态词向量，"screen"在任何上下文里都是同一个向量；LSTM把整句→单向量，丢失了方面词信息 |
| "F1是什么意思？" | Macro F1，三个类别（正/负/中）各自算F1再取平均，比Accuracy更能反映不平衡数据上的真实表现 |
| "训练多久？" | LSTM约3分钟（18个epoch），BERT约10分钟（1个epoch就最佳），之后过拟合 |

## 老师可能的追问 & 回答

| 追问 | 回答要点 |
|------|----------|
| "LSTM输入是什么维度？" | batch_size × max_seq_len(85)，词向量300维 |
| "BERT为什么要加[SEP]？" | 两个[SEP]标记Sentence Pair边界，配合Segment Embedding区分句子和方面词 |
| "为什么用macro F1？" | 数据可能类别不均衡，macro F1给每个类别相同权重，更能反映真实表现 |
| "怎么防止过拟合？" | Dropout(0.1)+Early Stop(patience=5)，BERT训练时1个epoch就最佳 |
| "怎么部署到服务器？" | 换debug=False，用gunicorn/uwsgi代替Flask自带服务器，加nginx反向代理 |
| "代码里最难的bug？" | transformers 5.x的BertModel输出从tuple变成了OrderedDict，迭代返回key而不是value，导致dropout收到字符串报错 |
