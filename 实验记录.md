## 四、推理测试

### 4.1 测试用例

**句子**："The screen is great but the battery is poor."（屏幕很好但电池很差）

### 4.2 LSTM 推理结果

| 方面词 | 预测结果 | Negative | Neutral | Positive | 是否正确 |
|--------|----------|----------|---------|----------|----------|
| screen | Positive | 0.29 | 0.08 | **0.62** | ✅ |
| battery | Positive | 0.29 | 0.08 | **0.62** | ❌ 应为Negative |

> **分析**：基础 LSTM 模型将整个句子编码后直接分类，**未显式利用方面词信息**，因此对同一句子中不同方面词给出相同结果。这是该模型的根本局限——它无法区分"对屏幕好评"和"对电池差评"。

### 4.3 BERT_SPC 推理结果

| 方面词 | 预测结果 | Negative | Neutral | Positive | 是否正确 |
|--------|----------|----------|---------|----------|----------|
| screen | Positive | 0.26 | 0.04 | **0.70** | ✅ |
| battery | Negative | **0.65** | 0.05 | 0.30 | ✅ |

> **分析**：BERT_SPC 将句子和方面词拼接为 `[CLS] 句子 [SEP] 方面词 [SEP]`，通过 Segment Embedding 区分句子和方面词，因此模型能感知到"当前分析的是哪个方面"。**这正是 BERT 大幅领先 LSTM 的关键原因。**

## 五、模型分析

### LSTM 特点
- **结构**：Embedding → LSTM → 取最后隐状态 → 全连接层 → 3分类（仅25行代码）
- **优势**：模型轻量（<1M参数），训练快（~3min），不依赖大预训练模型
- **劣势**：GloVe 是静态词向量，无法表示一词多义；基础 LSTM 未充分利用方面词信息
- **适用场景**：资源受限环境、对推理速度要求高的场景

### BERT_SPC 特点
- **结构**：预训练 BERT → Pooler → Dropout → 全连接层 → 3分类
- **优势**：12层 Transformer 编码器，在海量语料上预训练，能捕捉深层语义和上下文
- **劣势**：模型大（110M参数），推理需要 GPU，首次加载慢
- **适用场景**：追求高准确率、有 GPU 资源的场景

## 六、训练过程记录

### LSTM 训练日志

| 阶段 | 观察 |
|------|------|
| Epoch 0 | loss 从 1.11 降至 0.96，val_acc=0.6034 |
| Epoch 1-3 | loss 在 0.64-0.85 波动，val_acc 逐步升至 0.6505 |
| Epoch 4-8 | loss 降至 0.59-0.67，val_acc 达到峰值 0.6834 (Epoch 9) |
| Epoch 9-13 | 最佳模型出现在 Epoch 13: val_acc=0.6850, val_f1=0.6336 |
| Epoch 14-18 | 验证集指标不再提升，Epoch 18 触发 Early Stop |
| 最终测试 | test_acc=0.6850, test_f1=0.6336 |

**观察**：训练集准确率最高接近 80%，验证集在 65-68% 之间波动 → 存在轻微过拟合。

### BERT 训练日志

| 阶段 | 观察 |
|------|------|
| Epoch 0 | loss 从 1.16 降至 1.01，val_acc=0.6661 |
| Epoch 1 | loss 从 0.93 降至 0.62，**val_acc 跃升至 0.7868（最佳）** |
| Epoch 2-4 | loss 继续下降至 ~0.20，但 val_acc 下降至 0.73-0.74 → 过拟合 |
| 最终测试 | test_acc=0.7868, test_f1=0.7474（使用 Epoch 1 权重） |

**观察**：BERT 仅需 1 个 epoch 就达到最佳效果，之后迅速过拟合——大模型在小数据集上的典型表现。

## 七、问题与解决方案

| # | 问题 | 原因 | 解决方案 |
|---|------|------|----------|
| 1 | `tokenizers` 安装失败 | Python 3.14 太新，无预编译 wheel | 升级到 `transformers 5.13.0`，使用 `--only-binary :all:` |
| 2 | `pack_padded_sequence` 报错 `lengths must be CPU tensor` | PyTorch 2.11 不再接受 GPU tensor 作为 lengths | 在 `dynamic_rnn.py:65` 加 `.cpu()` |
| 3 | `LCF_BERT` 导入失败 | transformers 5.x 中路径变更 | 改为 `transformers.models.bert.modeling_bert` |
| 4 | BERT `dropout()` 报错 `must be Tensor, not str` | transformers 5.x 的 BERT 输出变为 OrderedDict，迭代返回 key 而非 value | 改用 `outputs.pooler_output` 属性访问 |
| 5 | HuggingFace 直连超时 | 国内网络限制 | 设置 `HF_ENDPOINT=https://hf-mirror.com` |
| 6 | GloVe 下载慢 | Stanford 服务器限速 | 用户手动下载 |
| 7 | Flask 启动 `UnicodeEncodeError` | Windows GBK 编码无法输出 ✓ | 设置 `PYTHONIOENCODING=utf-8`，print 改用英文 |

|------|--------|----------|----------|---------|----------|
